# OpenCalc HarmonyOS — 一句话自然语言增强需求

> 迁移已完成，进入增强开发阶段。以下需求均可用自然语言描述，覆盖从清晰到模糊、从简单到复杂的全谱系，
> 适合给客户演示"一句话需求驱动开发"的能力。

## 三、按演示场景推荐组合

### 场景 A：基础增强（5分钟演示）

适合：展示快速迭代能力

```
1. 加百分号按钮
2. 实时预览结果
3. 记忆功能 M+/MR
4. 科学计数法自动切换
5. 历史记录搜索
```

### 场景 B：AI理解力（5分钟演示）

适合：展示AI对模糊需求的理解和拆分能力

```
模糊需求: "让计算器更智能一点"
  → AI拆分为:
    1. 实时预览（不需要按等号）
    2. 自动纠正常见输入错误（如连续两个运算符）
    3. 结果智能格式化（大数科学计数法，小数保留合适位数）
    4. 表达式联想（输入sin时建议sin/cos/tan完整列表）
```

### 场景 C：技术深度（8分钟演示）

适合：展示复杂功能的实现能力

```
1. 程序员模式（二进制/十六进制 + 位运算）
2. 变量存储和引用
3. 函数图像绘制
4. 语音输入 + 语音播报
```

### 场景 D：生活化（5分钟演示）

适合：接地气，任何人都能理解

```
1. 小费计算器
2. 单位换算
3. 二维码分享
4. 剪贴板自动计算
```

## 五、客户演示策略

### 演示流程建议

```
1. 开场（1分钟）
   "这是一个从Android迁移到鸿蒙的计算器。迁移已完成，功能完全对齐。
    现在我们可以用一句话描述需求，由AI来驱动开发。"

2. 清晰需求演示（2分钟）
   现场说："加一个百分号按钮"
   → 展示AI理解 → 生成代码 → 编译运行 → 按钮可用

3. 模糊需求演示（3分钟）
   现场说："让计算器更智能一点"
   → AI自主拆解为4-5个具体子功能
   → 展示分析过程
   → 挑选1个实现（实时预览）

4. 进阶挑战（3分钟）
   现场说："支持二进制和十六进制转换"
   → 切换到程序员模式
   → 展示BIN/HEX/DEC切换

5. 收尾（1分钟）
   "我们在15分钟内演示了：
    - 1个模糊需求（AI拆解）
    - 2个清晰需求（快速实现）
    全部从一句话自然语言出发。"
```

### 如果只有一个需求...

选 **#2 实时预览结果**。

理由：
- 对用户体验的改变最直观（不用按等号就能看到结果）
- 技术实现简单（1行监听+1行计算）
- 演示效果好（输入过程中结果实时跳动）
- 客户立刻能理解价值
