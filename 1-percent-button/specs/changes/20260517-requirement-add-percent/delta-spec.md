# 需求SPEC: 百分号按钮

## 系统背景
> OpenCalc HarmonyOS 计算器已内建百分比计算引擎（getPercentString），但缺少 UI 入口。用户无法通过按钮触发百分比输入，只能手动输入 % 字符。

## 需求描述
| 需求编号 | 需求名称 | 需求描述 | 优先级 |
|---------|---------|---------|:--:|
| REQ-001 | 百分号按钮 | 计算器按钮网格增加 % 按钮，点击后表达式追加 %，计算时自动按百分比语义求值 | P0 |

## 场景用例
### 场景 SC-001: 百分比加法
#### 前置条件: 计算器处于基础模式
#### 主成功路径
1. 用户输入 "50"
2. 用户点击 "+"
3. 用户输入 "10"
4. 用户点击 "%"
5. 用户点击 "="
6. 显示结果为 "55"
#### 扩展路径（异常处理）
无异常路径

### 场景 SC-002: 百分比乘法
#### 前置条件: 计算器处于基础模式
#### 主成功路径
1. 用户输入 "200"
2. 用户点击 "×"
3. 用户输入 "15"
4. 用户点击 "%"
5. 用户点击 "="
6. 显示结果为 "30"

## 验收标准 (AC)
| # | 验收标准 | 优先级 | 验证方式 |
|---|---------|:--:|:--:|
| AC1 | % 按钮在按钮网格第 1 行可见 | P0 | 实测 |
| AC2 | 点击 % 后表达式区显示 % 字符 | P0 | 实测 |
| AC3 | "50+10%=" 计算结果为 55 | P0 | 实测 |
| AC4 | "200×15%=" 计算结果为 30 | P1 | 实测 |
| AC5 | 原有功能不受影响（加减乘除正常） | P0 | 回归测试 |

## 边界条件
| 场景 | 输入 | 预期行为 |
|------|------|---------|
| % 前无数字 | "%" 直接按等号 | 表达式错误 |
| 连续 %% | "50+10%%" | 表达式错误 |

## 假设和约束
| # | 约束类别 | 约束描述 | 对设计的影响 |
|---|---------|---------|------------|
| 1 | 布局 | % 放在首行 AC 右侧，16% 宽 | 首行由 4 按钮变为 5 按钮 |
