# 百分号按钮 需求提案

## 1. 需求概述
### 1.1 功能名称: 百分号按钮
### 1.2 需求类型: requirement-add
### 1.3 需求描述: 加一个百分号按钮，输入50+10%自动算出55
### 1.4 使用场景: 用户在计算器中输入含百分号的表达式（如折扣计算、百分比加法），按下等号得到正确结果
### 1.5 预期效果: 用户点击 % 按钮后，表达式追加 % 字符；计算时自动按百分比语义求值

## 2. 需求解析报告（rq-parse 输出）

### 2.1 意图分类
- **类型**: requirement-add
- **一句话概括**: 在计算器 UI 上增加一个百分号按钮，使已有的百分比计算逻辑可通过 UI 触发

### 2.2 领域映射
- UI/交互: 按钮网格增加 % 按钮
- 状态管理: 无变更（% 通过 onOp 追加到 expression 字符串）

### 2.3 影响层次
| 层次 | 变更类型 | 说明 |
|------|---------|------|
| View | modify | CalculatorPage.ets 按钮网格增加 % 按钮 |
| ViewModel | none | onOp 已支持任意字符串追加 |
| Model | none | 无数据模型变更 |
| 数据层 | none | 无持久化变更 |
| 基础设施 | none | Expression.ets 的 getPercentString() 已实现完整百分比语义 |

### 2.4 关键实体
- **页面实体**: CalculatorPage（已有）
- **事件实体**: onOp('%')（已有 onOp 方法可处理）

### 2.5 模糊点
| # | 类别 | 描述 | 严重程度 |
|---|------|------|---------|
| 1 | 交互模糊 | % 按钮放置位置 | important |

**⚠️ Assumption**: 将 % 按钮放在第一行（AC/%/(/)/÷），作为 5 按钮行，各按钮宽度 16%。

### 2.6 复杂度评估
- **等级**: S
- **影响模块数**: 1（CalculatorPage.ets）
- **架构变更**: 否
- **数据模型变更**: 否
- **理由**: Expression.ets 的 getPercentString() 已实现完整百分比语义（含 +/* 上下文展开）。只需 1 行新增按钮 + 调整首行布局。改动 <5 行。

## 3. 需求澄清（rq-clarify 输出）
### 3.1 已澄清的关键决策
| # | 问题 | 决策 | 来源 |
|---|------|------|------|
| 1 | % 按钮放在哪里 | 第一行 AC 右侧，5 按钮布局（16% 宽） | ⚠️ Assumption |

### 3.2 功能边界
- **包含**: UI 上新增百分号按钮，点击追加 % 到表达式
- **不包含**: 修改百分比计算逻辑（已实现）

|------|---------|-----------|

### 4.2 代码依赖分析
